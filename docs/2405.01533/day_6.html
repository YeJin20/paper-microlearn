방법론

Counterfactual checklist 속살 — 규칙 자동판정과 GPT-4의 분업

왜 raw 3D를 GPT-4에 그대로 넣지 않는지, 그리고 어떻게 rule + LLM 분업으로 풀어내는지 메커니즘 단위로 이해.
5일차에 본 파이프라인의 심장부, counterfactual checklist를 오늘은 더 깊이. 문제 설정: GPT-4에 multi-view 이미지 + 3D 박스 + 차선 좌표를 한꺼번에 던지면? 페이퍼는 '3D 관계를 효과적으로 이해하지 못한다, 객체가 많아질수록 더 심해진다'고 명시합니다. OmniDrive의 해법은 2단 분업이에요. 1) Rule-based 자동 판정: 객체 충돌, 도로 경계 이탈, 신호 위반처럼 명확한 카테고리는 nuScenes 3D detection + OpenLane-v2 lane topology로 코드가 직접 체크. 2) High-level decision으로 추상화 후 GPT-4: 미묘한 traffic rule은 trajectory를 '좌회전 + 가속' 같은 결정 텍스트로 변환한 뒤 GPT-4가 안전성과 사유를 추론. 예시 — 'simulated decision: Moderate Speed, Left Turn' + lane assignment 정보가 들어가면 checklist가 'Out of drivable area: True'를 자동 출력하고, GPT-4는 'opposite-flow lane으로 진입해 맞은편 차량과 충돌 위험'이라는 자연어 사유를 생성. 작지만 중요한 트릭 하나 — 'object as close' 정의는 trajectory와의 최소 거리 < 10m, 향후 3초 이내. attention 후보를 이 규칙으로 좁힙니다. 체크 — 여러분 task에서 '규칙으로 자동 판정 가능한 부분'과 'LLM에 맡겨야 하는 미묘한 부분'을 어떻게 가르시겠어요?
