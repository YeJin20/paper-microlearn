큰 그림 — Q&A 생성 파이프라인

데이터 생성 전체 흐름(키프레임→시뮬→체크리스트→GPT-4→QA→HITL) 큰 그림 잡기
오늘은 컴포넌트 디테일 말고 흐름만 잡습니다. 네 단계예요. (1) Planning-oriented keyframe selection: nuScenes에서 의미적·행동적 다양성을 두 번의 K-means로 추려냄. (2) Counterfactual checklist & prompt design: 시뮬 궤적 위에 rule-based 검사(충돌·도로 경계·신호)와 GPT-4 추론(고차원 의사결정)을 조합. (3) Q&A generation: 4가지 conversation type(scene description / attention / counterfactual reasoning / decision-planning)을 GPT-4가 일관되게 생성. (4) Human-in-the-loop QA로 prompt를 다듬은 뒤 대규모 iterate. 비유하자면, 요리 레시피를 소량으로 시제품 만들고 맛 보정한 다음 공장으로 보내는 구조예요. 시제품 단계에서 다양성이 보장돼야 공장 라인이 균질해집니다. 자가점검: 만약 우리 scene editing 파이프라인에 human-in-the-loop을 단 하나의 단계에만 박을 수 있다면, 키프레임 선택 / 체크리스트 / Q&A 검증 중 어디가 가장 ROI가 클까요? 내일부터 각 단계를 한 칸씩 들여다봅니다.
