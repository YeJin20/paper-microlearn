방법론

전체 파이프라인 한 장 — key-frame, counterfactual, checklist, human-in-the-loop

데이터 생성 파이프라인의 4단계 흐름을 빅픽처로 보여주고, 디테일 학습 전 reader가 전체 구조를 잡도록 함.
오늘은 OmniDrive 데이터 생성 파이프라인 전체 그림만. 디테일은 내일부터. 흐름은 4단계입니다. 1) Planning-oriented key-frame 선택: nuScenes 프런트 이미지의 CLIP 임베딩으로 K-means → 20% 선택, 다시 미래 trajectory로 K-means → 200개 클러스터 중심. 멈춤·직진·좌회전·우회전·U-turn·가감속·정속까지 운전 행동을 포괄. 2) Counterfactual 시뮬레이션: 위 카테고리별 가상 trajectory를 각 장면에 투영. 3) Rule-based checklist + GPT-4 reasoning: 충돌·도로 경계·신호 위반은 3D 어노테이션 + OpenLane-v2 lane topology로 자동 판정, 미묘한 케이스는 high-level decision으로 추상화 후 GPT-4가 안전성 reasoning. 4) Human-in-the-loop quality assurance: prompt와 checklist를 반복 튜닝, 통과 후 대규모 생성. 비유하자면 한 장면에 대해 '8가지 다른 미래'를 일부러 굴려보고, 각각의 결과를 라벨로 만드는 trajectory-level augmentation입니다. 운전 교습 영상을 한 장면당 8버전으로 늘리는 셈. 핵심은 'GPT-4에 raw 3D를 그대로 넣지 않는다'는 점이에요. 머릿속에서 한 번 돌려보세요 — 여러분 도메인 데이터로 똑같이 이 4단계를 적용한다면 1단계 key-frame 선택 기준은 무엇이 될까요?
