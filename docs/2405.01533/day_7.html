방법론

Omni-L 아키텍처 — 2D VLM에 부드러운 3D 안경을 씌우기

LLaVA 계열 위에 3D PE를 0 초기화로 얹는 Omni-L의 설계 철학과 학습 안정성 trick을 이해.
오늘부터 OmniDrive-agent의 두 백본 아키텍처. 먼저 Omni-L (LLaVA 계열). 설계 철학은 단순합니다 — '2D VLM을 그대로 가져와 3D로 살짝 확장하자.' 구조: - 비전 인코더 EVA-02-L로 multi-view 이미지 N장의 feature Fm 추출 - 각 패치에 3D positional encoding Pm 더함 (단, 학습 안정성을 위해 PE 가중치는 0으로 초기화) - Linear–GELU–Linear MLP projector로 LLM 토큰 공간(LLaMA2-7B의 4096-dim)에 정렬 - LLaMA2-7B가 텍스트 생성 핵심 트릭은 'PE를 0으로 시작'이에요. 학습 초기엔 LLaVA pretrained 표현이 그대로 살아 있고, 학습이 진행되면서 점진적으로 3D 정보를 흡수합니다. 마치 LLaVA에 부드러운 3D 안경을 씌우는 셈. 학습은 2단계 — 2D-Pretraining 후 3D-Finetuning. 두 단계 모두 contrastive/matching loss 없이 텍스트 생성 loss만 씁니다. 머릿속 실험 — 만약 PE를 무작위 초기화로 두면 어떻게 될까요? 페이퍼는 학습 안정성을 위해 0 초기화를 명시합니다. 즉 pretrained VLM 표현을 망가뜨리지 않는 게 핵심이에요. 내일은 정반대 진영, 3D perception 스택 위에 LLM을 얹는 Omni-Q.
