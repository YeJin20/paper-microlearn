오늘의 학습 · DAY 2
배경

선행 NeRF가 깨진 지점

동적 NeRF·AV NeRF 계열이 어떤 가정에서 운전 로그에 무너졌는지를 진단합니다.
어제 '단일 traversal 4D'가 본질적 어려움임을 합의했죠. 그럼 기존 NeRF 계열은 왜 이 벽 앞에서 무너졌을까요. 네 갈래로 정리됩니다. 첫째, Nerfies/HyperNeRF 같은 deformation 기반은 모든 관측을 첫 프레임에서 만든 canonical space로 워핑해 매핑한다고 가정합니다. 그러나 운전 장면에서 객체는 프레임 중간에 시야에 진입했다가 빠져나가므로 단일 canonical space로 모일 수가 없습니다. 둘째, D²NeRF는 monocular 동적 분해를 시도했지만 야외에서 hyperparameter에 과도하게 민감하고 capacity가 부족해 정적·동적 분리에 실패합니다(Table 1a에서 24.35 PSNR로 가장 낮음). 셋째, NSG·UniSim은 객체별 GT bounding box와 트래킹에 의존해 동적 객체를 따로 모델링합니다 — 성능은 나오지만 라벨 비용으로 인해 규모가 막힙니다. 넷째, SUDS는 multi-traversal 가정을 두고 pre-trained optical flow에 강하게 의존하며, flow warping loss가 비싸고 cycle-consistency 필터링이 깨지면 최적화가 발산합니다. 마지막으로 scene flow 쪽 NSFP는 각 타임스텝마다 작은 네트워크를 매번 다시 최적화하느라 사실상 실시간 사용이 불가능합니다. 즉 '자기지도 + 단일 traversal + flow도 직접 학습'을 동시에 만족하는 방법이 없었다는 게 빈자리입니다. 내일은 EmerNeRF가 이 빈자리를 어떤 한 문장의 아이디어로 채우는지부터 봅니다.

Trigger Type B
Canonical-space deformation 가정
Nerfies/HyperNeRF가 모든 시점의 관측을 첫 프레임 기준의 '정칙 공간(canonical space)'으로 deformation 매핑해 다시 NeRF로 학습한다는 설계입니다. 인물 얼굴 같은 닫힌 토폴로지에서는 잘 동작하지만, 운전 장면처럼 객체가 enter/exit하며 canonical에 존재하지 않을 수도 있는 토폴로지에서는 매핑 자체가 정의되지 않아 무너집니다. '항상 시작 프레임에 등장한다'는 암묵 가정이 도시 주행에서는 거의 위반됩니다.
Pre-trained optical flow 의존성
SUDS·DynIBaR 같은 동적 NeRF는 외부 optical flow 모델(RAFT 등) 출력을 flow warping loss로 주입합니다. 문제는 (1) cycle-consistency로 잘못된 flow를 걸러내지 못하면 최적화가 발산하고, (2) flow warping 비용이 매우 커서 학습이 느려지며, (3) flow 모델 자체의 도메인 편향이 그대로 NeRF에 전이된다는 점입니다. EmerNeRF가 'flow도 emergent'를 노린 이유가 정확히 이 의존성을 끊기 위해서입니다.

객관식 Q1
HyperNeRF·Nerfies가 운전 로그에서 실패하는 핵심 가정은?
객체가 enter/exit하는 운전 장면에서는 canonical space에 존재하지 않는 객체가 흔하므로 deformation 매핑 자체가 정의되지 않습니다.
객관식 Q2
NSG와 UniSim의 공통 약점은?
두 방법은 객체별 분리를 GT 라벨에 의존하므로, 데이터 양이 늘어날수록 라벨 비용이 비례해 증가합니다. SUDS와는 다른 종류의 의존성입니다.
주관식
본인 팀의 NeRF/3DGS 파이프라인이 '외부 모델 출력'에 의존하는 지점은 어디인가요? 그 의존성을 self-supervision으로 대체하면 어떤 비용이 사라지고 어떤 위험이 생길까요?
힌트
외부 모델 의존은 종종 학습 시작점을 안정화해주지만 도메인 시프트의 통로가 되기도 합니다. EmerNeRF는 후자를 끊는 쪽을 택했습니다.
꼭 짚고 가야 할 것
외부 신호 없이 학습이 부트스트랩되려면 '서로 강화하는 두 신호'(여기서는 분해와 flow)가 필요합니다. 단일 신호만으로 self-supervised를 하면 보통 collapse합니다.
