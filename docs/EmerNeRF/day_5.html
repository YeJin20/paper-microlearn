전체 파이프라인 지도

EmerNeRF의 표현·헤드·렌더링·손실을 한 흐름으로 펼쳐 큰 그림을 잡습니다.
오늘은 method 지도 한 장을 머릿속에 그립니다. EmerNeRF는 세 개의 hash-grid 필드를 씁니다. 정적 $S = H_s(x)$, 동적 $D = H_d(x, t)$, flow $V = H_v(x, t)$. 쿼리 $(x, t)$가 들어오면 정적·동적에서 각각 feature·density $(g_s, \sigma_s), (g_d, \sigma_d)$ 를 뽑고, $V$가 출력하는 forward·backward flow $v_f, v_b$로 인접 timestep의 동적 feature를 가중합 $g'_d = 0.25 g_d(x+v_b, t-1) + 0.5 g_d(x, t) + 0.25 g_d(x+v_f, t+1)$ 로 묶습니다. 그다음 네 개의 헤드가 갈라집니다. shared color head가 정적·동적 color $c_s, c_d$ 를 같은 가중치로 예측(정적의 dense supervision을 동적이 빌려쓰는 트릭), sky head는 ray 방향에서 sky 색 $c_{sky}$ 를, shadow head는 동적 객체의 shadow ratio $\rho \in [0,1]$ 를, feature head는 DINOv2 lifted feature $f$ 를 뽑습니다. 최종 색은 density-weighted blending $c = \frac{\sigma_s}{\sigma_s + \sigma_d}(1-\rho) c_s + \frac{\sigma_d}{\sigma_s + \sigma_d} c_d$ 로 합쳐지고, ray를 따라 volume rendering으로 적분됩니다. 학습은 RGB L2, sky BCE, shadow sparsity, dynamic density 페널티, cycle consistency, depth + line-of-sight, feature L2가 모두 한 번에 흐르는 end-to-end. 모델은 약 30M parameters, A100 1장에서 정적 33분, 동적 2시간, feature 추가 시 +5~15분이면 끝납니다. 큰 그림은 'hash-grid 3개 + head 4개 + 손실 7개를 한 backbone에 박았다'입니다. 내일부터 3일은 이 지도의 세 핵심 구역(분해 / flow / feature)을 한 구역씩 확대해서 봅니다.

Trigger Type B
Hybrid hash-grid 3-field 표현
EmerNeRF는 시간 독립 feature($H_s$, 정적)와 시간 의존 feature($H_d$, 동적), 그리고 flow($H_v$)를 모두 iNGP 스타일 multi-resolution hash grid로 표현합니다. 정적은 2^4~2^13 해상도 10 레벨, feature dim 4, 레벨당 max 2^20 entry. 동적은 거의 동일하지만 max 2^18. 이 분리는 (1) 시간에 무관한 정적 구조에 dense gradient를 모아 표현 효율을 높이고, (2) 동적 분기는 '꼭 필요한 곳'에서만 capacity를 쓰게 만들어, 라벨 없는 분해의 inductive bias로 작동합니다. 총 파라미터 약 30M로 StreetSurf보다 18M 작습니다.
Density-weighted static-dynamic blending
정적·동적 분기 모두에서 density가 나올 수 있으므로, 한 점의 최종 색은 $c = \frac{\sigma_s}{\sigma_s + \sigma_d}(1-\rho) c_s + \frac{\sigma_d}{\sigma_s + \sigma_d} c_d$ 로 두 분기를 density 비율로 섞습니다. 여기서 $\rho$ 는 shadow head 출력으로, 동적 객체가 정적 배경 위에 드리우는 그림자 강도를 조절합니다. 이 형태는 '정적이 dominant일 땐 정적 색, 동적이 살아 있을 땐 동적 색'을 부드럽게 보간하면서, 그림자라는 정-동 결합 효과까지 한 식에 담는다는 점이 우아한 부분입니다.

객관식 Q1
EmerNeRF의 shared color head가 정적·동적 분기에 동시에 적용되는 가장 큰 이유는?
동적 관측은 sparse하므로, 정적 분기에서 풍부하게 들어오는 RGB gradient를 같은 color head를 통해 동적도 함께 학습합니다.
객관식 Q2
Density-weighted blending 식에서 shadow ratio $\rho$ 의 역할은?
$(1-\rho)c_s$ 형태로 정적 분기의 색을 modulate함으로써, 동적 객체가 정적 배경에 드리우는 그림자를 표현합니다.
주관식
본인이 운영 중인 reconstruction 파이프라인에서 '정적·동적 분기에 공유할 수 있는 헤드'는 어디가 있을까요? 공유하면 동적 sparse supervision 문제가 줄어들 모듈이 있나요?
힌트
Geometry head, semantic head, BRDF/material head 같이 시간에 약한 의존성을 갖는 출력이 후보가 됩니다.
