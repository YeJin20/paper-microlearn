Emergent scene flow

Flow field가 어떻게 라벨 없이도 정확히 학습되는지 식과 ablation으로 풀어봅니다.
오늘은 EmerNeRF에서 가장 마법 같은 부분, scene flow가 어떻게 'emergent'하는지를 봅니다. Flow field는 $V := H_v(x, t)$ 와 작은 MLPv로 구성되며 출력은 6-dim — forward $v_f$ 와 backward $v_b$. 핵심은 이 flow를 어떻게 supervise하느냐인데, 저자들은 어떤 explicit flow loss도 쓰지 않습니다. 대신 동적 feature를 인접 timestep과 묶는 temporal aggregation 식 $g'_d = 0.25 g_d(H_d(x+v_b, t-1)) + 0.5 g_d(H_d(x, t)) + 0.25 g_d(H_d(x+v_f, t+1))$ 한 줄이 모든 일을 합니다. 이 $g'_d$ 가 color/feature/shadow head로 흘러가 RGB reconstruction loss를 만들어내므로, gradient가 자연스럽게 flow까지 거슬러 올라옵니다. 왜 emergent가 일어날까? 저자들의 가설은 명확합니다 — '시간적으로 일관된 feature만 multi-frame aggregation에서 이득을 본다'. 잘못된 flow로 모은 feature는 reconstruction loss를 증가시키므로 손해를 보고, 옳은 flow는 보상을 받습니다. 자연 선택입니다. 보조로 cycle consistency $\mathcal{L}_{cycle}$ 가 forward $v_f$ 와 그 다음 프레임의 backward $v_b$ 사이 일관성을 stop-gradient를 끼워 가볍게 강제합니다. Ablation이 이 가설을 깔끔하게 입증합니다(Table C.1). 'no temporal aggregation' 또는 '인접 feature 그래디언트 정지' 설정에서는 flow Acc5가 각각 4.53%, 3.87%로 무너지지만, 디폴트는 89.74%입니다. 결과적으로 EPE3D 0.014m로 NSFP의 0.365m를 압도(약 26배), Acc5는 51.76→93.92%(약 42% 절대 향상). 라벨 없는 flow가 SOTA의 supervised baseline을 넘어선 거의 첫 사례라는 점에서, 이 한 줄의 aggregation 식이 논문 전체에서 가장 비싼 줄입니다. 내일은 flow와 무관한 또 다른 축, ViT feature 통합과 PE 문제로 갑니다.

Trigger Type A
Temporal feature aggregation (식 8)
동적 feature를 forward/backward flow로 인접 timestep과 가중합 $g'_d = 0.25 g_d(x+v_b, t-1) + 0.5 g_d(x, t) + 0.25 g_d(x+v_f, t+1)$ 하는 매우 단순한 모듈입니다. 가중치 0.25/0.5/0.25는 단순한 prior이고 학습되지 않습니다. 이 한 식이 세 일을 동시에 합니다 — (1) flow field가 scene reconstruction loss에 연결, (2) 인접 프레임 feature를 denoise해 동적 표현 안정화, (3) shared gradient를 통해 점 단위 feature가 시간적 이웃과 정보를 공유. ablation에서 이걸 빼면 flow가 0 근처로 collapse하므로 emergence의 직접 원인입니다.
Cycle consistency loss $\mathcal{L}_{cycle}$
Forward flow $v_f(x, t)$ 로 점을 t+1로 옮긴 뒤, 그 위치의 backward flow $v'_b(x+v_f, t+1)$ 가 원래 $-v_f$ 와 같아야 한다는 제약을 stop-gradient(sg)와 함께 평균제곱으로 부과: $\mathcal{L}_{cycle} = \frac{0.01}{2} E[(sg(v_f) + v'_b)^2 + (sg(v_b) + v'_f)^2]$ . Stop-gradient가 한쪽에만 들어가 학습이 한 방향으로만 흐르도록 비대칭을 만들고, 계수 0.01로 약하게 걸려 over-regularization을 피합니다. 보조 역할이며, 진짜 학습 신호는 어디까지나 aggregation을 통한 reconstruction loss입니다.

객관식 Q1
Temporal aggregation 식의 가중치 (0.25, 0.5, 0.25)는 어떻게 결정되는가?
논문은 단순한 가중합으로 고정해두며, 학습되지 않습니다. 단순함이 emergence를 막지 않는다는 점이 흥미로운 부분입니다.
객관식 Q2
Ablation에서 'no temporal aggregation' 설정 시 flow Acc5가 4.53%로 무너지는 이유는?
Aggregation을 빼면 flow는 어떤 손실에도 연결되지 않아 학습 신호가 0이 됩니다. Cycle만으로는 학습이 일어나지 않음을 의미합니다.
주관식
'단순한 가중합 한 줄로 라벨 없이 flow가 학습된다'는 결과가 본인 도메인의 어떤 모듈에 영감을 줄까요? 동등한 'self-supervised correspondence' 자리가 어디에 있을까요?
힌트
Stereo depth, optical flow, multi-frame BEV, point tracking 등 모두 'temporally consistent feature' 가설이 살아있는 자리입니다.
꼭 짚고 가야 할 것
Aggregation은 평균화 도구가 아니라 supervision의 통로입니다. 어디서 무엇을 aggregate하느냐가 곧 학습할 신호를 결정합니다.
