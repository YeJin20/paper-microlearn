데이터 파이프라인 분해

체크리스트→GPT-4→5종 Q&A 변환의 작동 원리
오늘은 OmniDrive의 심장인 반사실 Q&A 생성 파이프라인을 분해해 봅시다. 출발점은 전체 nuScenes 궤적에서 뽑힌 200개 군집의 시뮬레이션 궤적입니다. 각 키프레임에 정지·좌회전·U턴 같은 후보 행동을 던져 보고, 결과를 두 갈래로 평가합니다. ①규칙 기반 항목 — 객체 충돌, 도로 경계 충돌, 신호 위반은 nuScenes 3D 박스와 OpenLane-V2의 centerline·topology로 결정적 검증. ②규칙 외 항목 — 그 외 교통 규칙은 시뮬레이션 궤적을 "lane assignment, lane-change" 같은 high-level decision으로 변환한 뒤 GPT-4에 위임. 두 신호를 합쳐 "이 좌회전은 안전한가?" 같은 판단을 만듭니다. 프롬프트는 3종 입력으로 구성됩니다: ⓐ 캡션(다중뷰 6장을 stitched 전·후 2장으로 만들어 GPT-4가 먼저 묘사), ⓑ 시뮬레이션 결정·궤적, ⓒ 전문가 결정·궤적 + close objects(궤적과 3초·10m 이내). 출력은 Scene Description / Attention / Counterfactual Reasoning / Decision making & Planning / General conversation 5종 Q&A입니다. 핵심 트릭 하나: 이미지와 장면 정보를 동시에 입력하면 GPT-4가 이미지를 무시하는 경향이 강합니다. 그래서 캡션을 먼저 강제 생성하게 만들어 OCR·롱테일 객체 인식을 보존했어요. 작은 디테일이 데이터 품질을 좌우합니다.

객관식 Q1
close object의 정의로 옳은 것은?
논문은 'minimum distance < 10m within next 3 seconds'로 close object를 정의한다.
객관식 Q2
다중뷰 이미지를 먼저 캡션으로 변환한 뒤 추가 정보를 주는 설계 이유는?
캡션 선행 → OCR·롱테일 객체 인식을 보존, 동시 입력 시 이미지 무시 문제를 회피한다.
주관식
