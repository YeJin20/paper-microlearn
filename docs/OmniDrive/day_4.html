개요

비슷한 데이터셋과의 위치

DriveLM·LingoQA·Reason2Drive 대비 OmniDrive의 자리
운전 LLM 데이터셋 지형도를 펼쳐 보면 몇 갈래로 갈립니다. 인식·QA 중심(nuScenes-QA, NuPrompt)은 객체 카운팅·위치 묘사에 강하지만 계획·추론은 약해요. 명령·내비형(Talk2Car, LaMPilot)은 자연어 명령→행동 매핑에 집중하죠. 설명 가능성형(BDD-X, Reason2Drive)은 ego 행동의 사후 해명에 초점을 둡니다. 가장 가까운 친척은 DriveLM과 LingoQA입니다. DriveLM은 GVQA(Graph VQA)로 perception→prediction→planning 의존성을 그래프로 다루고, LingoQA는 비디오 QA에 반사실 질문을 일부 도입했어요. 하지만 둘 다 2D QA에 머무르거나, 3D 궤적 단위의 반사실 분석은 본격적으로 다루지 않습니다. OmniDrive는 그 사이를 정확히 메웁니다. 시뮬레이션 궤적→체크리스트→GPT-4를 거쳐 3D 좌표 위에서의 반사실 추론을 다루죠. 트레이드오프는 두 가지: ①데이터 규모는 nuScenes의 696 scenes로 제한적이고, ②단일 차량 시뮬레이션이라 다른 에이전트의 reactive 거동은 빠집니다. 반면 보상은 분명해요. 같은 nuScenes 위에 풍부한 reasoning supervision을 얹어, planning과 explainability를 동시에 평가할 수 있게 됩니다. 내일은 이 데이터셋을 학습한 두 에이전트(Omni-L/Q)의 전체 파이프라인을 펼쳐 봅니다.

객관식 Q1
OmniDrive가 DriveLM·LingoQA와 가장 명확히 차별화되는 지점은?
DriveLM은 2D QA에, LingoQA는 비디오 QA에 머문 반면 OmniDrive는 3D 궤적-반사실 통합을 강조한다.
객관식 Q2
현재 OmniDrive 설계에 명시된 한계 중 사실인 것은?
Limitations 섹션이 'does not yet consider reactions from other agents'라고 직접 명시한다.
주관식
지형도 위에서 봤을 때, dual-system AD를 위한 데이터셋은 어떤 자리에 있어야 할까요? 본인이라면 OmniDrive와 어떻게 합치거나 분리하시겠어요?
힌트
System 1(빠름·짧은 horizon)과 System 2(느림·긴 horizon·언어 설명)가 요구하는 데이터 특성을 따로 떠올려 보세요.
꼭 짚고 가야 할 것
dual-system을 노린다면 'reasoning trace'와 'fast control trajectory'를 분리해 생성하는 데이터셋 설계가 필요합니다. OmniDrive는 그중 reasoning 쪽 절반을 담당하는 좋은 후보예요.
