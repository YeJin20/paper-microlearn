Omni-Q 깊이 보기

Q-Former + 3D PE + perception query의 조합
Omni-Q는 "기존 3D perception 스택을 VLM에 끼워넣자"는 쪽 시도입니다. 토대는 StreamPETR식 sparse 3D 쿼리 + BLIP-2의 Q-Former 디자인. 핵심 메커니즘은 두 쿼리의 분업입니다. carrier query Qc는 언어-시각 정렬을 책임지고, detection query Qd는 3D 객체 검출을 책임져요. self-attention에서 둘이 정보를 섞고: (Q, K, V) = ([Qc, Qd], [Qc, Qd], [Qc, Qd]) 그다음 cross-attention으로 다중뷰 특징 + 3D positional encoding Pm을 흡수합니다: (Q, K, V) = ([Qc, Qd], Pm + Fm, Fm) 여기서 Pm이 핵심입니다. 3D PE 덕분에 carrier query가 단순한 의미 정렬을 넘어 기하학적 prior를 함께 끌어오죠. 마지막으로 Qd는 3D 객체 검출 손실로 직접 감독되고, Qc는 MLP를 거쳐 LLaMA2-7B의 4096차원 토큰으로 매핑됩니다. 머릿속 시도: 이 구조의 장점과 단점을 동시에 그려 보세요. 장점은 충돌 같은 3D 정밀 태스크에 강하다는 것 — collision Recall 72.6%, drivable area Recall도 안정적. 단점은 언어 능력 자체가 LLaVA 계열보다 약해 종합 counterfactual AP/AR이 떨어지고 CIDEr도 낮다는 것. perception을 깊게 박을수록 language를 깎아먹는 트레이드오프가 또렷이 드러납니다. 즉, Omni-Q는 "3D 인식 보강"이라는 구체적 무기를 얻는 대신 "언어 일반화"라는 비용을 치릅니다.

perception 보강의 대가로 language 능력이 약해지는 패턴이 Tab. 4·5에서 확인된다.
주관식
dual-system 관점에서, Omni-Q처럼 perception을 깊이 박은 구조는 System 1과 System 2 중 어느 쪽에 더 가깝다고 보시나요? 왜?
힌트
3D 충돌 검출이 강한 것은 빠른 회피 결정에 필요하고, 언어 reasoning이 약한 것은 긴 추론 체인에 약함을 뜻합니다.
꼭 짚고 가야 할 것
Omni-Q의 perception 강화는 System 1의 reactive 능력과 닮았고, Omni-L의 언어 능력은 System 2의 reasoning 친화도와 닮았어요. 한쪽 시스템에 맞춰 projector를 고를 수 있다는 게 dual-system 설계의 단서가 됩니다.
