배경

기존 시도는 왜 막혔나

open-loop 평가의 ego status 과적합과 2D VLM의 3D 결손
지금까지 운전 LLM-agent 연구들이 부딪힌 벽은 크게 두 가지입니다. 첫째, open-loop 평가의 함정. UniAD·VAD 같은 모듈형 end-to-end 모델은 nuScenes에서 좋은 L2 점수를 받지만, BEV-Planner 연구가 폭로하듯 자기차량 상태(ego status)에 심하게 과적합돼 점수가 부풀려져요. ego 속도·방향만 잘 외워도 충돌률이 뚝 떨어지니, 실제로 장면을 이해했는지 분별이 안 되는 거죠. 게다가 로그-리플레이 환경 자체가 단순해서, 모델이 전문가 궤적에 그대로 오버핏하기 쉽습니다. 둘째, VLM의 3D 결손. LLaVA·BLIP-2 같은 2D VLM은 강력한 추론 능력을 갖지만, 다중 카메라 영상을 3D 공간으로 정렬하는 능력은 빈약합니다. 이미지·3D 박스·차선 좌표·장면 정의를 GPT-4에 그냥 우겨넣어도, 객체 간 관계나 차량의 주행 상태를 제대로 읽지 못하죠. 장면 요소가 많아질수록 더 망가집니다. DriveLM, LingoQA, Reason2Drive 같은 운전 LLM 데이터셋도 반사실 질문을 부분적으로 다루지만, 3D 궤적 단위로 통합한 사례는 드물어요. '풍부한 supervision'과 '3D 정렬'을 동시에 푼 프레임워크가 비어 있던 셈입니다. 잠깐 자가점검: open-loop L2 점수가 낮다고 해서 '주행 능력이 좋다'고 단정할 수 있을까요?

객관식 Q1
BEV-Planner 연구가 폭로한 open-loop 평가의 핵심 약점은?
Ego-MLP/BEV-Planner는 ego 상태만으로도 SoTA에 근접한 점수가 나오는 현상을 통해 평가의 신뢰성 문제를 지적했다.
객관식 Q2
2D VLM을 3D 운전 장면에 그대로 적용했을 때 가장 두드러진 결손은?
논문은 다중 정보를 한꺼번에 넣을수록 GPT-4가 이미지를 무시하고 관계 추론도 실패하는 패턴을 명시한다.
주관식
open-loop L2/Collision 지표가 ego status에 오염된다면, 본인 연구에서 dual-system 스위처의 성능을 평가할 때 어떤 보조 지표를 추가해야 할까요?
