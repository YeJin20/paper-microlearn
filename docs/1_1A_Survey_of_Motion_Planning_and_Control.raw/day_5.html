방법론

전체 파이프라인 조감도

deep dive 전, 결정 계층과 차량 모델 두 축으로 전체 파이프라인을 개관
오늘은 deep dive 전에 '지도'를 펼칩니다. 전체 파이프라인은 두 축으로 봅니다. (축1: 결정 계층, Fig II.1) 목적지 → 경로계획(도로망을 가중그래프로 보고 최단경로; 대륙규모 네트워크도 전처리 후 ms 단위로 푼다) → 행동결정(차선유지·차선변경·교차로 양보 등 국소 과업 선택; 유한상태기계나 (PO)MDP로 타 참가자 의도의 불확실성 처리) → 모션계획(국소 충돌회피·동역학 가능 궤적 생성) → 피드백제어(추종오차 실시간 보정). 각 층의 출력이 다음 층의 입력이 되는 직렬 구조죠. (축2: 모델, Section III) 계획과 제어는 모두 차량 모델 위에서 돕니다. 저속·주차엔 미끄러짐 없음을 가정한 기구학 단일트랙(bicycle) 모델이면 충분하고, 고속·미끄러운 노면처럼 관성효과가 커지면 타이어 힘과 슬립을 다루는 동역학 모델이 필요합니다. 즉 모델 선택이 곧 계획·제어의 난이도를 정합니다(Day 1의 트레이드오프!). 비유하면 계층은 '여행 일정'(대륙→도시→골목→발걸음)이고, 모델은 '내 몸 상태'(평지냐 빙판이냐)입니다. 둘이 맞아야 안전하죠. 흥미로운 비대칭도 하나—경로계획은 정적 그래프라 거의 즉시 풀리지만, 모션계획은 연속·미분제약·동적 장애물 때문에 가장 무거운 병목입니다. 다음 3일간 모델 → 모션계획 → 제어 순으로 한 층씩 파고듭니다.
참고 그림
Figure II.1 — 의사결정 계층 다이어그램
상단 목적지 입력 → Route Planning → Behavioral Layer(Lane Following·Change Lanes·Negotiate Intersection·Parking 등 박스) → Motion Planning → Local Feedback Control로 내려가는 수직 파이프라인. 좌측엔 도로망데이터·인지된 에이전트 입력, 우측엔 각 단계 출력(웨이포인트→모션사양→기준궤적→조향/가감속 명령).
Trigger Type B
기구학 단일트랙(bicycle) 모델
앞뒤 바퀴를 강체 링크로 잇고 평면 운동·바퀴 무슬립을 가정한 가장 기본 모델. 후륜 기준 미분제약은 $\dot{x}_r = v_r\cos\theta,\ \dot{y}_r = v_r\sin\theta,\ \dot{\theta} = \tfrac{v_r}{l}\tan\delta$ (여기서 $l$=축거, $\delta$=조향각). 왜 쓰나: 두 바퀴를 하나로 묶어 상태를 3차원($x,y,\theta$)으로 줄이면서도 비홀로노믹 거동을 포착한다. 저속·주차·도심처럼 관성효과가 작을 때 유효하다. 변형으로 $v_r$ 고정 시 Dubins 차, 전후진 허용 시 Reeds-Shepp 차. 함정: 순간적 조향각 변화를 허용하므로 연속 곡률이 필요하면 조향각/조향률을 상태로 증강해야 하고, 한계조작·저마찰에선 깨진다.
행동결정의 불확실성 처리 (FSM vs POMDP)
행동층은 '지금 무엇을 할지'(차선유지/변경, 교차로 양보 등)를 결정한다. DARPA Urban Challenge 대다수 팀은 운전 맥락을 상태로, 인지 결과를 전이조건으로 하는 **유한상태기계(FSM)**+휴리스틱을 썼다. 그러나 실제 도심은 타 참가자 의도가 불확실해, 이를 명시적으로 다루려 **MDP/POMDP**로 정식화하고 근사해법(예: 보행자 의도를 잠재상태로)을 쓴다. 직관: FSM은 '규칙표'라 명확하지만 예외에 취약(brittle)하고, POMDP는 부분관측 불확실성을 원리적으로 다루나 계산이 비싸다. 의도 예측엔 GMM·가우시안 프로세스 회귀 등 학습 기법도 등장한다.

순차적 파이프라인의 단계를 연결해 서술할 때
객관식 Q1
기구학 단일트랙 모델만으로 충분한 상황으로 가장 적절한 것은?
기구학 모델은 무슬립 가정이 유효한 저속·주차·도심에서 적합하다. 관성·타이어 슬립이 커지는 고속·저마찰에선 동역학 모델이 필요하다.
객관식 Q2
행동결정 층에서 POMDP를 도입하는 주된 이유는?
도심 주행의 핵심 난점은 타 참가자 의도가 불확실하고 부분관측이라는 점이다. POMDP는 이 불확실성을 명시적으로 다루는 틀을 제공한다.
주관식
경로계획은 대륙 규모도 ms로 푸는데 모션계획은 가장 무거운 병목이다. 두 문제의 본질적 차이는 무엇인가?
힌트
한쪽은 정적 가중그래프 위 최단경로(전처리로 가속 가능), 다른 쪽은 연속공간·미분제약·동적 장애물을 동시에 다룬다.
꼭 짚고 가야 할 것
경로계획과 모션계획은 같은 '계획'이라도 복잡도 클래스가 다르다. 정적 이산 그래프 최단경로는 전처리로 ms에 풀리지만, 연속·미분제약·동적 모션계획은 근본적으로 더 어려운 문제다.
