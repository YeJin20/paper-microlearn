점·Gaussian·초기화

기하 모델(배경+객체)과 LiDAR 기반 초기화 깊이 보기
첫 번째 deep dive — 점·Gaussian 기하 모델. 각 점은 3D Gaussian 1개. 파라미터는 평균 μ, 회전 R, 스케일 S, opacity α, SH 계수, semantic logit. 공분산은 Σ = R·S·Sᵀ·Rᵀ (Eq.1) — R은 unit quaternion, S는 대각 성분만. 이렇게 분해하는 이유는 임의 행렬이 *유효한 공분산*이 되지 못할 위험을 막기 위해서. 배경 모델: μ_b, R_b를 world 좌표계에서 직접 학습. semantic은 M차원 logit β_b. 객체 모델: μ_o, R_o는 차량 *local 좌표계*에 살다가, 매 프레임 트래킹 포즈 (R_t, T_t)로 world로 변환 (Eq.2): μ_w = R_t·μ_o + T_t, R_w = R_t·R_o. semantic은 1차원 scalar(차량 클래스 id) — 렌더 시 one-hot으로 펼쳐 배경과 합쳐요. 초기화가 결정적입니다. 3D GS의 SfM 점은 텍스처 없는 도시 거리에 무력. 대신 ego LiDAR 누적 점을 쓰고 이미지에 투영해 색을 입힙니다. 객체 박스 안 LiDAR 점이 2K 미만이면 박스 안 8K 점 랜덤 샘플링으로 대체. 학습 중 박스 *바깥*으로 자라난 객체 Gaussian은 가지치기 — occlusion 영역으로의 누설 차단. 내일은 외형: 4D SH로.

객관식 Q1
공분산 Σ를 직접 학습하지 않고 R·S·Sᵀ·Rᵀ로 분해해 R, S만 학습하는 이유는?
임의 행렬은 유효 공분산이 아닐 수 있음. 회전(quaternion)·스케일로 분해하면 항상 유효 공분산이 보장된다.
객관식 Q2
박스 안 LiDAR 점이 2K 미만일 때 박스 안 *8K 점을 랜덤 샘플링*해 초기화하는 이유는?
점이 부족하면 splatting 품질이 무너짐. 박스라는 강한 prior로 안전한 위치에 초기 점을 뿌리는 셈.
