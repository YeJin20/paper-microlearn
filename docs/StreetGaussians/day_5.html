분해→외형→합성→학습 4단계로 큰 그림 잡기
오늘은 method 큰 그림. 파이프라인을 4단계로 보면: ① 분해 (Decomposition) — 한 시퀀스를 '정적 배경 포인트클라우드 1개' + '차량 N개의 포인트클라우드 N개'로 쪼갭니다. 각 점은 3D Gaussian(위치·공분산·opacity·SH·semantic)을 들고 있어요. ② 외형 (Appearance) — 배경 점에는 표준 Spherical Harmonics. 차량 점에는 4D SH — 시간에 따라 SH 계수가 변하도록 Fourier 계수로 펼친 모델(차가 움직이면 빛·반사가 바뀌니까). ③ 합성 + 렌더 — 시점 t에서 객체 점을 (R_t, T_t)로 world 좌표계로 옮기고 배경과 concat. 2D 평면에 EWA splatting으로 투영(Eq.4)해 α-blending(Eq.5). 멀리 있는 하늘은 별도 cubemap(1024 해상도)으로 채웁니다. ④ 학습 (Training) — 30000 iter, Adam, RTX 4090 한 장. color · depth(LiDAR) · sky · semantic · reg loss 동시 최적화(Eq.7). 트래킹 포즈도 학습 가능한 ΔR·ΔT로 잡아 노이즈를 흡수. 내일부터 3일간 deep dive: ① 점·Gaussian 모델, ② 4D SH, ③ pose optimization.

Trigger Type A
EWA splatting
3D Gaussian을 2D 이미지 평면에 투영할 때 사용하는 anisotropic 가우시안 필터 근사. 점 단위 differentiable rasterization을 가능케 한다.
α-blending
정렬된 점들을 앞→뒤로 누적해 색을 합성하는 컴포지션 규칙. C = Σ c_i α_i ∏(1 − α_j).

Street Gaussians가 *하늘* 영역만 별도 cubemap으로 처리하는 핵심 이유는?
Gaussian은 유한 위치에 정의됨 — 무한대 거리의 하늘을 점으로 채우는 건 비효율적이고 부정확하다.
