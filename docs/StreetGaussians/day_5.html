전체 파이프라인 조망

깊이 들어가기 전, 표현→합성→렌더링→학습으로 이어지는 전체 지도를 큰 그림으로 제시
딥다이브 전에 '지도'를 먼저 펼칩니다. Street Gaussians의 파이프라인은 네 토막으로 읽으면 됩니다. 1) 표현(Representation): 장면 = 배경 점군 1개 + 객체 점군 N개. 모든 점은 공통 속성을 가집니다 — 위치 $\mu$, 불투명도 $\alpha$, 스케일 $S$, 회전 $R$(공분산), 색을 내는 SH 계수, 그리고 semantic logit. 배경은 여기까지. 객체 점군은 두 가지를 더 얹습니다: 최적화 가능한 tracked pose와 시간 의존 외관을 내는 4D SH. 2) 합성(Composition): 렌더 시각 $t$에서 4D SH로 그 순간의 색을 복원하고($z_{m,l}(t)$), 객체 점군을 tracked pose로 world 좌표계로 변환($\mu_w = R_t \mu_o + T_t$)한 뒤, 배경 점군과 그냥 이어 붙입니다(concatenate). NeRF처럼 복잡한 합성 ray-marching이 필요 없다는 게 핵심 우아함이죠. 3) 렌더링(Rendering): 합쳐진 점군을 2D로 투영해 $\alpha$-blending. 색 대신 semantic logit을 넣으면 semantic map도, 깊이도 같은 틀로 나옵니다. 먼 하늘은 Euclidean 가우시안으로 부적합해 별도 cubemap으로 처리해 합성합니다. 4) 학습(Training): 가우시안 + 하늘 cubemap + tracked pose를 색·깊이·하늘·semantic·정규화 다항 손실로 동시 최적화하고, LiDAR 점군으로 초기화합니다. 오늘은 이 4토막의 연결만 머리에 넣어두세요. 내일부터 토막을 하나씩 해부합니다.
참고 그림
Figure 2 — Street Gaussians 전체 개요도
scene representation(배경+객체 모델, geometry: position/rotation/opacity/scale, dynamic appearance: Fourier coeff×time basis×SH basis) → composition → point-based rendering → decomposition으로 이어지는 데이터 흐름 도식. 객체만 dynamic appearance를 갖는 비대칭 구조에 주목.
Trigger Type B
배경 모델 vs 객체 모델의 비대칭 설계
두 모델은 $\alpha$, $S$, opacity 등 다수 속성을 공유하지만 세 곳에서 갈립니다. (1) 좌표계: 배경은 world, 객체는 local 좌표에 $\mu_o, R_o$를 두고 tracked pose로 world로 변환($\mu_w=R_t\mu_o+T_t,\ R_w=R_tR_o$). (2) 외관: 배경은 정적 SH, 객체는 시간 의존 4D SH. (3) semantic: 배경은 $M$차원 logit 벡터 $\beta_b$, 객체는 차량 클래스를 나타내는 1차원 스칼라 $\beta_o$. 왜 비대칭인가: 움직이는 건 차량뿐이므로 '동역학 비용'을 객체에만 부과해 효율을 챙기는 설계입니다.
Concatenation 기반 합성(rendering by composition)
기존 compositional NeRF가 객체별 ray-marching을 합성해야 했던 것과 달리, Street Gaussians는 객체 점군을 world로 변환한 뒤 배경 점군과 단순히 이어 붙여 하나의 점군으로 만들고 한 번에 splatting 합니다. 이 단순함이 곧 속도이자 편집성입니다: 점군을 더하고 빼는 집합 연산만으로 객체 삽입·삭제·교체가 됩니다. 주의할 점: 올바른 가시성(depth ordering)과 $\alpha$-blending이 합쳐진 점군 전체에서 일관되게 처리되어야 자연스러운 가림이 나옵니다.

객관식 Q1
Street Gaussians의 합성이 compositional NeRF보다 단순한 본질적 이유는?
explicit 점군이라 객체를 world로 옮겨 이어 붙이는 집합 연산만으로 합성이 끝난다. 객체별 ray-marching 합성이 필요 없다.
객관식 Q2
배경 모델과 객체 모델이 다르게 설계된 항목으로 옳은 조합은?
객체는 local 좌표+tracked pose, 4D SH, 1차원 semantic 스칼라를 쓰고 배경은 world 좌표, 정적 SH, M차원 logit을 쓴다. 동역학 비용을 객체에만 부과한 비대칭 설계다.
주관식
동적 비용을 '객체에만' 부과하는 이 비대칭 설계는, 우리가 다루는 장면(예: 흔들리는 가로수, 펄럭이는 현수막)에서 언제 깨질 위험이 있을까?
힌트
이 설계는 '배경은 정적, 움직이는 건 추적된 강체 차량뿐'이라는 가정 위에 서 있습니다. 그 가정을 벗어나는 동적 요소를 떠올려 보세요.
꼭 짚고 가야 할 것
비대칭 설계의 효율은 '배경=정적' 가정에 의존한다. tracker로 잡지 않는 비강체·미세 동적 요소(나무, 깃발, 보행자)는 배경에 흡수되며 잔상이나 흐림으로 새어 나온다 — 적용 도메인에서 이 가정의 성립 여부를 먼저 점검해야 한다.
