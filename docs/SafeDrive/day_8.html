PwNC와 TwDAC의 수식과 의도
FRNet은 SWNet이 만든 128개 정제된 world 각각에 대해 두 fine-grained score를 계산해 최종 trajectory를 결정합니다. (1) Pair-wise No Collision (PwNC) — 각 planning query를 같은 world의 모든 instance query와 pair로 concat해 MLP+sigmoid로 통과시켜, ego의 j번째 trajectory와 i번째 agent 사이의 시점별 collision-free 확률 $p^{i,j}_{pwnc}(h) \in [0,1]$ 를 예측. trajectory 전체 PwNC는 $$P^{j}_{PwNC} = \prod_{i=1}^{N} \prod_{h=1}^{H} p^{i,j}_{pwnc}(h).$$ N개 agent × H시점 곱이 핵심입니다. 단 한 (agent, 시점)에서 확률이 낮으면 전체 점수가 빠르게 0으로 갑니다 — collision의 catastrophic 성격을 그대로 모델링한 것. Figure 4(b)가 보여주는 'i번째 agent의 future box 위에 PwNC 점수가 빨강~초록으로 칠해진' 시각화가 이 (agent, 시점) 좌표계의 정직한 그림입니다. (2) Time-wise Drivable Area Compliance (TwDAC) — $F_{BEV}$ 에서 ConvNeXt-v2로 static prior $F^{static}_{BEV}$ 를 추출하고 segmentation head로 drivable area·lane centerline을 예측. 정제된 미래 ego 위치를 reference로 trajectory-guided deformable attention을 돌려 시점별 점수 $p^{j}_{twdac}(h)$ 를 얻습니다. 다만 학습된 점수는 경계 부근에서 흔들리므로, BEV segmentation map $M_{Driv}$ 에서 ego 박스의 9개 key point (중심·네 꼭짓점·네 변 중점)을 bilinear interpolation으로 샘플링해 곱합니다: $$P^{j}_{TwDAC} = \prod_{h=1}^{H}\left(p^{j}_{twdac}(h) \times \prod_{k=1}^{9} M^{j}_{Driv}(h,k)\right).$$ MLP가 못 잡는 'box 모서리가 차선 밖' 같은 경계 사례를 hard geometric prior로 보강하는 구조. Ablation Table 12에서 temporal과 interpolation 두 요소가 각각 ~0.3 DAC씩 기여하고 합쳐 DAC 99.0 에 도달. (3) 통합 — PwNC, TwDAC, ProposalNet의 scene-level 점수들을 log-probability space에서 weighted log-sum해 최고 점수 trajectory를 선택. log-space는 곱셈 catastrophic 신호를 안정적 gradient로 다루기 위한 표준 트릭입니다.

Trigger Type C
Pair-wise No Collision (PwNC)
ego의 j번째 trajectory와 i번째 agent 사이의 collision-free 확률을 시점별로 예측: $p^{i,j}_{pwnc}(h) \in [0,1]^H$. 전체 점수는 모든 agent와 시점에 대한 곱 $P^{j}_{PwNC} = \prod_{i,h} p^{i,j}_{pwnc}(h)$. 곱 구조의 의미: 한 (agent, 시점)이 낮으면 전체가 0에 가까워져, catastrophic 사건의 비대칭성을 자연 표현. holistic scoring과 비교해 '어디서 위험이 발생했는지' 분해 가능. 함정은 모든 agent를 동일 가중으로 곱하므로 미검출 agent의 risk는 표현 불가 — perception에 직접 의존하는 reasoning. 또 곱셈은 학습 시 log-space에서 다뤄야 gradient가 안정.
Time-wise DAC with 9-keypoint interpolation
TwDAC는 두 신호를 곱한다. (i) MLP 기반 시점별 점수 $p^{j}_{twdac}(h)$ — 학습된 soft 점수, (ii) ego 박스의 9 key point(중심·4 꼭짓점·4 변 중점)에서 BEV segmentation map $M_{Driv}$ 를 bilinear interpolation으로 샘플링한 hard prior. 식: $P^{j}_{TwDAC} = \prod_{h}(p^{j}_{twdac}(h) \prod_{k=1}^{9} M^{j}_{Driv}(h,k))$. 왜 9점인가: 중심만 보면 박스 모서리가 차선을 밟는 경우를 놓치고, 너무 많이 보면 redundancy. 9 keypoint는 'box geometry의 minimal 표현'으로 정착. Ablation에서 temporal과 interpolation 각 +0.5/+0.3, 합쳐 DAC 99.0.

객관식 Q1
PwNC가 $P^{j}_{PwNC} = \prod_{i,h} p^{i,j}_{pwnc}(h)$ 형태의 곱셈 결합을 택한 가장 중요한 동기는?
곱 구조는 worst-(agent, timestep)을 보전. collision의 'one-strike-out' 성격을 점수 함수에 직접 반영하는 디자인 선택.
객관식 Q2
TwDAC가 MLP 점수와 9 key point interpolation을 곱하는 이유로 가장 적절한 것은?
MLP의 soft 점수와 segmentation map의 geometric prior가 서로 상호 보완. 박스 모서리가 차선 밖에 살짝 걸리는 경계 사례는 후자가 잡아주고, 멀리서의 추세는 전자가 잡아줌.
주관식
본인의 planner가 마지막으로 만든 unsafe trajectory 하나를 떠올려, '$\prod_{i,h} p^{i,j}_{pwnc}(h)$ 의 곱셈 분해'를 적용한다면 어느 agent의 어느 시점에서 점수가 무너졌을지 머릿속으로 시뮬레이션해 보세요. 그 진단 결과를 데이터/모델/objective 중 어디에서 고치겠습니까?
힌트
곱셈 분해는 attribution을 가능하게 합니다. 'i번째 agent의 t+5에서 0.1'이라는 결과가 나오면 그건 perception, prediction, planning 중 어느 단계의 문제인지를 좁혀줍니다.
꼭 짚고 가야 할 것
Fine-grained safety score는 단순 평가가 아니라 진단 도구. 'PwNC가 어디서 무너지는가'를 데이터 augmentation·loss reweighting·perception 강화 중 어떤 lever로 푸느냐가 본인 시스템의 다음 한 수를 결정한다.
