방법론

SWNet — Sparse World를 짓다

Sparse World Generator + World Interaction Module의 동작
SWNet은 128개 safety-aware planning query 각각에 대해 독립적인 Sparse World를 만들고, 그 world 안에서 ego와 surrounding agent의 미래를 함께 정제합니다. (1) Sparse World Generator — instance query와 그 trajectory를 $K'=128$번 복제해 각 planning query와 concat. 결과적으로 'planning query 1개 + 모든 surrounding instance query'가 한 world. 이게 128개 만들어집니다. dense grid 대신 'ego + N agents'라는 finite set만 들고 가는 것이 sparse라는 이름의 본질입니다. 후보별 독립 world라 'ego가 직진하면 어떻게 되는가' vs 'ego가 좌회전하면 어떻게 되는가'가 분리된 시뮬레이션 공간에서 평가됩니다. (2) World Interaction Module — 한 world 안에서 두 단계 attention이 흐릅니다: $$\bar{\mathcal{W}} = \mathrm{World\text{-}SelfAttn}(\mathcal{W}),$$ $$\hat{\mathcal{W}} = \mathrm{FFN}(\mathrm{Deform\text{-}Attn}_{traj}(\bar{\mathcal{W}}, \mathcal{T}_{world}, F_{BEV})).$$ 첫 self-attention은 모든 dynamic agent 사이의 pair-wise interaction 을 명시 모델링합니다 — ego의 의도까지 query 형태로 들어가 있으므로 'ego가 이 trajectory를 택했을 때 옆 차가 어떻게 반응할지'가 자연스럽게 표현됩니다. 두 번째 trajectory-guided deformable attention은 각 entity의 예측 미래 trajectory $\mathcal{T}_{world}$ 를 따라 $F_{BEV}$ 에서 spatio-temporal feature를 더 모읍니다. Key insight: ego와 surrounding agent의 trajectory가 동시에 refine 되므로 mutual consistency가 보장됩니다. 'ego가 양보하면 옆 차가 끼어든다' 같은 reactive 패턴이 한 world 안에서 일관되게 표현되는 것. Ablation Table 10에서 surrounding-agent motion refinement를 빼면 PDMS가 91.6에서 90.9 로 떨어지는 0.7 점 차가 정확히 이 효과입니다. SWNet은 디코더 4개 layer로 쌓여 점진적으로 정제됩니다(ProposalNet은 2 layer). 내일은 이 정제된 world 위에서 FRNet이 어떻게 fine-grained risk를 계산하는지 봅니다.

Trigger Type B
World Interaction Module (Self-Attn + Deform-Attn)
한 sparse world 안에서 두 단계 attention을 순차 적용. 첫 단계 self-attention은 ego planning query를 포함한 모든 entity query 사이의 **pair-wise interaction** 을 표현해 'ego가 이 경로를 택했을 때 i번째 agent가 어떻게 반응할지'를 한 번에 모델링. 두 번째 trajectory-guided deformable attention은 각 entity의 미래 경로 $\mathcal{T}_{world}$ 위에서 $F_{BEV}$ 를 따라 spatio-temporal feature를 추가 sampling. 식은 $\hat{\mathcal{W}} = \mathrm{FFN}(\mathrm{Deform\text{-}Attn}_{traj}(\bar{\mathcal{W}}, \mathcal{T}_{world}, F_{BEV}))$. 결과적으로 ego·agent의 미래가 **공동 refine** 되어 mutual consistency가 강제되고, 이게 reactive 시나리오에서의 성능 차이를 만든다. 잘 알려진 함정은 self-attention이 quadratic이지만 sparse라 N이 작아 부담이 적다는 점.

객관식 Q1
SWNet의 World Interaction Module에서 self-attention이 'ego planning query'까지 함께 입력으로 받는 것이 핵심인 이유는?
ego query가 self-attention 안에 들어가 있어야 'ego가 이 경로를 택했을 때 i번째 agent가 어떻게 반응할지' 같은 conditional reactive 패턴이 표현됨. 이게 sparse world의 본질적 차별점.
객관식 Q2
Ablation Table 10에서 surrounding-agent motion refinement를 추가했을 때의 PDMS 변화는?
Table 10에 따르면 surrounding-agent motion refinement까지 더하면 90.9에서 91.6으로 0.7 PDMS 상승. 이게 reactive interaction을 명시 모델링한 직접적 이득.
주관식
본인 시스템의 multi-agent prediction 모듈이 'ego의 plan에 conditioning되지 않은' 형태라면, SafeDrive식 'ego query를 self-attention에 함께 넣는' 디자인을 도입할 때 어떤 새 supervision 신호가 필요할까요?
힌트
reactive ground-truth가 없으면 conditional behavior를 학습하기 어렵습니다. 시뮬레이터·log replay가 어떤 형태의 신호를 줄 수 있을지 떠올려 보세요.
꼭 짚고 가야 할 것
Conditional prediction은 dataset이 '같은 장면에서 ego의 plan을 바꿨을 때 agent가 어떻게 달라지는가'를 담아야 학습 가능하다. CARLA·NAVSIM처럼 reactive simulator가 없으면 이 신호는 약하고, log-replay만으로는 이 효과를 다 끌어낼 수 없다 — sim2real 인프라 설계 의사결정과 직결되는 지점.
