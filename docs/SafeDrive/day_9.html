NAVSIM·Bench2Drive 결과 + 핵심 ablation 해석
숫자는 어디까지 paper에 있고, 무엇을 enable하는지 봅니다. NAVSIM (open-loop) — SafeDrive는 PDMS 91.6 (이전 SOTA GoalFlow 90.3 대비 +1.3), NC 99.5, DAC 99.0, TTC 97.2, EP 84.3, Comfort 100. 12,146 test 중 61건 collision (0.5%) 으로 collision-critical 메트릭에서 분리된 격차를 만듭니다. 확장 메트릭 EPDMS 87.5 도 SOTA로, DDC 99.7, TLC 99.9, LK 97.5, HC 98.2까지 rule-based 준수가 균형 잡혀 있습니다. Bench2Drive (closed-loop) — Driving Score 66.77, Success Rate 42.40%. 직전 SOTA DriveTransformer(DS 63.46) 대비 +3.31점 — open-loop가 잡지 못하는 reactive·long-horizon 시나리오에서 sparse + fine-grained 결합이 일관되게 동작함을 보여주는 가장 강한 신호입니다. Ablation 핵심 (Table 4·5·6) — Diff head만 88.6 → +Prop 89.1 → +SL Eval 90.1 → +SWNet 또는 +FRNet 각 90.9 → 둘 다 91.6. SWNet/FRNet 각 단독 효과는 0.8씩, 함께 적용하면 추가 0.7. 더 흥미로운 비교(Table 5): BEV world + scene-level safety만으로는 90.9에서 멈추지만, Sparse world + scene-level + fine-grained는 91.6. 즉 fine-grained 안전 reasoning은 dense 표현 위에서는 이득이 거의 없고, sparse world와 결합해야 0.7 PDMS가 나옵니다 — 'sparse가 fine-grained의 enabler'라는 논문 주장의 정량 증거. PwNC·TwDAC만 따로 보면 각 +0.6/+0.5, 함께 +0.7 (Table 6). Latency — BEVFormer + SafeDrive 67ms (DiffusionDrive 50ms 대비 +17ms). fine-grained reasoning의 직접 비용이지만 안전성 격차 대비 합리적입니다. 또 perception backbone을 TransFuser로 바꾸면 다른 baseline의 perception이 떨어지면서도 SafeDrive의 PDMS 이득은 TransFuser·BEVFormer 모두에서 유지되는 robustness가 Table 9에서 확인됩니다. 자기 체크: 만약 ablation에 'BEV world + fine-grained safety'만 따로 있었다면 어떤 결과가 예상됩니까?

Sparse가 Fine-grained의 Enabler 라는 정량 증거 (Table 5)
Table 5는 world representation × safety granularity의 2×3 비교. (BEV, scene-level) PDMS 90.9, (Sparse, scene-level) 90.9 — 즉 단순히 world를 sparse로 바꾸는 것만으론 이득 없음. (Sparse, scene + fine-grained) 91.6. 차이 +0.7이 'sparse가 있어야 fine-grained이 의미 있다'는 핵심 주장. 직관: dense grid는 (agent, 시점) 단위 신호를 표현할 자리가 없어, 같은 fine-grained head를 얹어도 학습 신호가 grid 평균으로 평탄화. Sparse는 instance query라는 자연스러운 슬롯을 줘서 fine-grained 점수가 의미를 가짐. 함정: 이 결과는 reverse causation으로 읽으면 안 됨 — 'fine-grained이 sparse를 정당화'가 아니라 'sparse가 fine-grained를 가능하게'.

객관식 Q1
Table 5에서 (BEV world + scene-level safety)와 (Sparse world + scene-level safety)의 PDMS는 모두 90.9였습니다. 이 사실의 함의로 가장 정확한 것은?
Sparse는 fine-grained의 enabler. fine-grained 점수가 instance query라는 슬롯을 사용해야 sparse가 이득(90.9 → 91.6)을 낸다.
객관식 Q2
Bench2Drive에서 SafeDrive의 DS가 직전 SOTA DriveTransformer 대비 +3.31 향상된 사실이 open-loop NAVSIM 결과와 별도로 갖는 의미는?
Closed-loop는 자기 출력이 다음 입력에 영향을 미치는 비정상적 분포 — open-loop SOTA가 자동으로 따라오지 않음. +3.31 DS는 long-horizon 동안 안전 reasoning이 누적 이득을 만든다는 증거.
주관식
본인 시스템에서 NAVSIM 같은 open-loop 평가만 있고 Bench2Drive 같은 closed-loop가 없다면, sparse+fine-grained 디자인이 효과적인지를 어떻게 진단해야 할까요?
힌트
open-loop는 distribution shift가 누적되지 않아 일부 효과가 가려집니다. 어떤 추가 평가를 한 번 더 돌리면 closed-loop의 신호를 대신할 수 있을지 떠올려 보세요.
꼭 짚고 가야 할 것
Open-loop 단독 평가는 fine-grained safety의 catastrophic 곱셈 효과를 일부 가린다 — 한 번의 위반이 누적되지 않기 때문. 가장 가까운 대안은 (1) collision rate를 별도 reporting, (2) trajectory perturbation 후 평가, (3) closed-loop replay 시뮬레이션 — 세 가지가 보완적이다.
