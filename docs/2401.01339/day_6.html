배경 모델 — 가우시안 + semantic logit

정적 3D 가우시안에 semantic logit이 왜 같이 붙어있는지 (하늘 정규화·semantic 렌더·3D fusion)
배경 모델은 가장 단순한 쪽이에요. 기본은 3D Gaussian Splatting 그대로 — 각 점이 (μ, Σ=RSS^TR^T, α, SH 계수 z)를 가집니다. 거기에 추가된 한 가지가 semantic logit β_b ∈ R^M — 각 점이 M개 클래스에 대한 점수를 들고 있어요. 왜 색만 두지 않고 semantic까지 점에 박았을까요? 셋입니다. 1) 하늘 정규화. 3D 가우시안은 Euclidean 공간이라 무한 멀리의 하늘을 표현 못 해요. 그래서 Grounded SAM이 뽑은 sky mask로 alpha를 0 쪽으로 미는 BCE supervision을 걸어, 하늘 영역엔 가우시안이 자라지 못하게 막습니다. 그 자리에 cubemap이 들어가죠. 2) Semantic map 렌더링. 색 c 대신 logit β를 splatting하면 그대로 2D semantic map이 나옵니다. KITTI에서 mIoU가 입력으로 쓴 Video K-Net의 53.81을 넘어 58.81이 나왔어요 — 3D fusion이 2D 예측을 정제한 셈. 3) Scene 이해. 3D semantic이 일관되니 그림자 같은 ambiguous 영역에서도 분할이 안정적입니다. 자문 — semantic을 빼면 어떤 시각적 artifact가 어디서 가장 먼저 터질까요? 답은 하늘과 thin structure(가로등·표지판 봉) 근처입니다. cubemap을 못 받쳐주거든요.
