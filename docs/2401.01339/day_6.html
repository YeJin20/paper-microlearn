Gaussian 기하 모델 깊게 보기

배경·객체 모델의 기하 파라미터화와 좌표 변환
첫 번째 deep dive — Gaussian geometry입니다. 배경과 객체 모두 점마다 3D Gaussian을 갖습니다. 핵심 파라미터는 위치 μ, 공분산 Σ, 불투명도 α, SH 계수. 공분산은 직접 학습하면 양의정부호성을 잃기 쉬워서, 스케일 S와 회전 R로 분해합니다: Σ = R S Sᵀ Rᵀ. 객체가 특별한 이유는 좌표계가 로컬이기 때문입니다. tracker가 준 회전 Rₜ, 평행이동 Tₜ로 매 timestep마다 world로 변환: μ_w = Rₜ μ_o + Tₜ, R_w = Rₜ R_o. 이 분리 덕분에 한 객체의 점군은 시간이 흘러도 "차량 기준 좌표계"에서는 정지 상태로 학습됩니다. 비유하자면 회전목마 위의 말은 자기 자리에선 가만히 있지만 세상 좌표에선 빙글빙글 도는 셈이죠. 학습이 훨씬 쉬워집니다. 또 객체는 box 밖으로 점이 자라지 못하도록 확률밀도 기반 점 샘플링 → box 밖이면 prune하는 장치를 둡니다. 배경 점은 20m 스케일로 고정해 occlusion 영역으로 새는 걸 막죠. 이 작은 디테일이 decomposition 깔끔함을 결정합니다.

객관식 Q1
공분산 Σ를 직접 학습하는 대신 Σ = R S Sᵀ Rᵀ로 분해해 학습하는 핵심 이유는?
R/S 분해는 항상 양의준정부호 공분산을 보장해 최적화가 안정적이다.
객관식 Q2
객체 점군을 'world 좌표계'가 아닌 '로컬 좌표계'에서 학습하는 이점은?
로컬 프레임에서 점이 정지이므로 시간 변화는 pose에 흡수되고, 점의 기하 학습은 단순화된다.
