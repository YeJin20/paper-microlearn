다시 우리 일로 — scene editing 데이터 증강

explicit·instance 분리·tracker pose·4D SH가 합쳐져 가능해지는 편집 연산과 후속 연구, 그리고 자기 일에 끌어오는 자문
10일의 마지막. 지금까지 본 디자인 — explicit primitive, instance 분리, tracker pose 결합, 4D SH — 이 모두가 한 가지 능력으로 수렴합니다: scene editing. 논문이 직접 보여주는 편집 연산 • Rotation: 차량 heading 회전 (R_t에 곱하기) • Translation: 위치만 이동 (T_t 변경) • Swap: 한 차량의 가우시안 묶음을 다른 시퀀스에서 가져와 끼우기 표현 디자인이 자연스럽게 열어주는 축 • 차량별 시간 t를 어긋나게 두면 추월·끼어들기 counterfactual 생성 • 4D SH의 Fourier 계수 일부를 흔들면 조명·그림자 augmentation • semantic logit이 점에 박혀있으니 새 차량을 끼울 때 라벨이 자동 재투영 후속 흐름도 짧게 — DrivingGaussian은 incremental static + composite dynamic graph로 surround view를 확장했고, OmniRe는 보행자·자전거 같은 non-rigid instance까지 카테고리를 일반화합니다. 둘 다 Street Gaussians의 '분리된 explicit primitive' DNA를 그대로 이어받은 줄기예요. 마지막 한 가지, 머릿속으로 직접 그려보세요 — 본인의 현재 scene editing 파이프라인에서 어떤 한 부분을 Street Gaussians 표현으로 갈아끼우면 가장 먼저 만들고 싶은 augmentation은 무엇이고, 그 결과로 어떤 perception 모델의 어떤 약점을 메우고 싶은가요? 한 줄로 노트에 적어두면 다음 실험의 출발선이 됩니다.
