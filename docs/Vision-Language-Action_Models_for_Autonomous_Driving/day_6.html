Textual vs Numerical generator 심층
오늘은 End-to-End VLA를 그 안에서 두 갈래로 쪼갠다 — *출력이 텍스트 공간에 있는가, 숫자 공간에 있는가*. 같은 백본을 써도 이 선택이 시스템의 성격을 갈라놓는다. Textual Action Generator. 모델이 운전 결정을 *언어로* 생성한다. 두 하위 형태: - Meta-action: 'accelerate', 'change lane', 'stop' 같은 이산 심볼. DriveMLM이 LLM 출력을 modular planner의 behavioral state와 정렬해, 언어가 'intermediate planner' 역할을 하도록 만들었다. AlphaDrive는 GRPO[Shao et al. 2024]를 도입해 trajectory quality·decision correctness·format consistency 보상을 결합한 RL로 meta-action을 다듬는다. - Trajectory waypoint as text: 좌표를 그대로 텍스트 토큰으로 출력. DriveLM은 graph-structured VQA로 perception→prediction→planning을 연쇄 질의로 풀고, 마지막에 텍스트화된 waypoint를 뱉는다. EMMA(Gemini 기반)는 카메라+navigation command를 한 언어 파이프라인으로 통합. 장점: reasoning과 action이 한 sequence로 학습됨. 약점: 이산 토큰 ↔ 연속 control 사이 *quantization* 갭, 그리고 극단적으로는 'collapsed trajectory'(반복 토큰). Numerical Action Generator. 백본은 그대로 두고 *숫자를 직접 뽑는 메커니즘*을 붙인다. - Additional action head: BEVDriver(GRU+BEV), CoVLA-Agent(MLP), DriveGPT4-V2(token+MLP 결합), DSDrive(dual-head: waypoint head + reasoning head + 큰 VLM에서 distill). DriveMoE는 lane-following·overtaking 등 skill별 expert를 dynamic routing. ORION은 head를 deterministic MLP에서 diffusion으로 바꿔 multi-modal future 분포를 모델링. - Additional action tokens: AutoVLA가 대표. 연속 trajectory를 codebook으로 *이산화*해 reasoning token과 같은 autoregressive sequence에 끼워 넣는다. RL fine-tuning으로 중복 reasoning을 패널티 줘 token 효율을 높임. OpenDriveVLA는 2D/3D perception을 LM의 semantic space에 hierarchical alignment한 뒤 token으로 decode. Trade-off 요약. Textual은 해석성이 거의 공짜로 따라오지만 정밀도가 떨어진다. Numerical은 control과 호환되지만 supervised data가 많이 필요하고, 토큰화 시 quantization artifact가 fine-grained control을 갉는다. 내일은 'VLM이 행동에 직접 개입하지 않고 *옆에서 가이드*'하는 Dual-System으로 넘어간다.

Action Token Discretization (AutoVLA식)
연속 trajectory $(x_i, y_i)$를 *codebook*으로 이산화해 LLM의 vocabulary에 추가 token으로 통합. reasoning과 action이 *동일 autoregressive sequence*로 생성되어 학습/inference가 단일 loss로 묶인다. 장점: 단일 next-token 손실로 reasoning ↔ action 정합 학습이 자연스러움. 단점: codebook 해상도가 fine-grained control 한계를 정함 — quantization artifact가 곡률 추종이나 미세 횡방향 제어에서 드러나기 쉽다. AutoVLA는 redundant reasoning에 RL penalty를 걸어 token 효율을 보강.
Group Relative Policy Optimization (GRPO) for VLA
DeepSeek 계열에서 도입된 RL 알고리즘으로, value model 없이 같은 prompt에 대한 *그룹 샘플 간 상대 advantage*로 정책을 갱신. AlphaDrive·Drive-R1·AutoVLA가 채택. driving에서는 **trajectory quality + decision correctness + format consistency** 같은 복합 보상을 하나의 group reward로 묶기 좋다. 장점: PPO 대비 value head 비용 절감, 복합 보상 통합 용이. 함정: reward shaping이 잘못되면 reasoning이 *사후 합리화*로 collapse — Drive-R1은 SFT-CoT 후 RL의 2단계로 이를 완화.

객관식 Q1
Textual action generator의 *근본적* 한계로 논문이 짚는 것은?
텍스트 공간에서 행동을 생성하면 연속 제어와 만날 때 quantization 갭이 본질적 한계 — 극단적 경우 collapsed trajectory까지 발생.
객관식 Q2
AutoVLA의 핵심 설계 선택은?
AutoVLA는 action token discretization으로 reasoning ↔ action을 동일 sequence에서 다루며, RL로 토큰 효율을 추가 최적화.
주관식
내가 textual head 모델을 prototype 한다면, quantization artifact를 가장 빨리 진단할 1개 metric은?
힌트
Open-loop L2도 좋지만 더 직접적인 건 *steering jerk*나 *yaw rate spike의 분포*. 사람 로그 분포 대비 꼬리가 두꺼우면 token 해상도가 부족한 신호다.
꼭 짚고 가야 할 것
Average L2/CR만 보면 quantization 문제가 가려진다. 미세 제어 quality는 jerk·yaw rate 같은 *고차 도함수의 분포*로 봐야 보인다.
