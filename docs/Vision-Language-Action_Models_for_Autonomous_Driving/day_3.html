개요

한 문장 핵심: VLA

VLA의 정식 정의·직관·기존 대비 우위
한 문장 요약. VLA는 멀티모달 입력 $x$(영상·LiDAR·언어·ego state)를 VLM 백본 $F(\cdot|\theta)$로 통합 표현으로 매핑한 뒤, action head $H(\cdot)$로 실행 가능한 행동 $a_t$를 내놓는 모델이다: $$a_t = H(F(x|\theta))$$ 수식은 단순해 보이지만 함의는 크다. 기존 VA에서 $F$는 ResNet 같은 visual encoder였다. VLA의 $F$는 수십~수백억 파라미터의 사전학습 VLM이고, 따라서 $\theta$ 안에 이미 'world knowledge'와 'language reasoning' prior가 들어 있다. action head $H$는 그것을 trajectory·control·텍스트 중 어떤 형태로든 뽑아낼 수 있는 어댑터다. 직관: System 2를 끼워 넣는다. Kahneman의 *Thinking, Fast and Slow*[2011]가 자연스러운 비유다. VA는 빠른 직관(System 1) 전용 — 반사적이지만 새로 보는 상황에 약하다. VLA는 거기에 느린 숙고(System 2) 를 결합한다. 비보호 좌회전 같은 상황에서 "대향 차량 속도와 거리, 보행자 의도, 좌회전 우선 규칙을 따져 본다" 같은 단계적 추론이 백본 안에서 token 단위로 펼쳐진다. 왜 이 접근이 직전 세대보다 나은가. ① 사전학습 활용 — long-tail 케이스에 대한 세상 지식을 처음부터 모으지 않아도 된다. ② 표현 자체에 reasoning 트레이스가 들어 있다 — CoT 출력으로 의사결정 근거를 직접 검증할 수 있다. ③ 자연어 인터페이스 — 운전자의 "좀 더 조심스럽게" 같은 추상 지시를 그대로 받는다. ④ action head를 갈아끼울 수 있다 — 같은 백본 위에 trajectory regression, action token, diffusion head를 자유롭게 얹는 모듈식 구조. 내일은 이 패러다임이 현실에서 어떻게 두 갈래(E2E vs Dual-System)로 분기했는지를 본다.
참고 그림
Figure 1 — Outline
Sensors·Language·Vehicle State 입력이 Model로 들어가 Trajectory·Action을 내놓는 좌측 다이어그램. VLA의 입력-처리-출력 구조를 가장 단순화한 그림으로, $x \rightarrow F \rightarrow H$ 수식 직관과 정확히 대응된다.
Trigger Type B
VLA 정식 형식화 $a_t = H(F(x|\theta))$
VLA의 세 요소: (i) **multimodal input** $x$ — 카메라·LiDAR·언어·ego state를 모은 heterogeneous tensor, (ii) **VLM backbone** $F(\cdot|\theta)$ — ViT 인코더 + LLM 디코더 + bridge로 구성된 사전학습 VLM, (iii) **action head** $H(\cdot)$ — language head(LH), regression(REG), trajectory selection(SEL), generation(GEN) 중 선택. 수식은 단순하지만 $\theta$에 사전학습 prior가 응축돼 있다는 점이 VA와 본질 차이다. *주의*: language를 입력으로 받는다는 사실과 출력 형식은 독립적이며, 출력은 텍스트일 수도 numeric trajectory일 수도 있다 — 이 분리가 Day 6의 분류 축이다.
Dual-Process(System 1/2) 영감
심리학의 dual-process theory(Kahneman)를 driving에 차용한 설계 철학. **System 1**(빠른 반사) = 기존 E2E planner / 전용 트라젝토리 모듈, **System 2**(느린 숙고) = VLM의 CoT reasoning. 둘을 하나의 네트워크에 통합하면 E2E VLA, 분리하면 Dual-System VLA가 된다. 이 메타포는 *latency 분리*와 *interpretability 분리*라는 공학적 결정을 자연어로 정당화해 주지만, 실제 인간 인지의 dual-process가 그대로 자율주행 architecture로 옮겨지는지는 별개 문제 — 영감일 뿐 증명은 아니다.

객관식 Q1
수식 $a_t = H(F(x|\theta))$에서 VLA가 VA와 갖는 *본질적* 차이를 만드는 부분은?
VA도 멀티모달일 수 있고 head 형태도 다양하다. 결정적 차이는 백본의 사전학습 — 거기서 long-tail 일반화와 reasoning 능력이 온다.
객관식 Q2
Dual-process 메타포에서 'System 2'에 대응되는 VLA 구성은?
느린 숙고(System 2)는 step-wise reasoning을 펼치는 VLM 백본에 해당. 다른 셋은 빠른 처리·기하 변환 모듈.
주관식
내가 푸는 문제에서 '$F$를 키우면 풀리는가, $H$를 갈아끼우면 풀리는가'?
힌트
Long-tail 인지 실패(perception miss)라면 $F$의 prior가 부족한 것이고, 인지는 되는데 action이 어색하다면 $H$의 표현력/inductive bias 문제다.
꼭 짚고 가야 할 것
VLA의 가치는 백본과 head를 *분리해서 갈아끼울 수 있다는 모듈성*에 있다. 자기 문제의 병목이 perception/reasoning인지 action 생성인지 먼저 진단해야 어디에 투자할지가 결정된다.
