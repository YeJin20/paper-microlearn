Dual-System VLA 내부

Explicit Guidance vs Implicit Transfer
Dual-System은 *VLM과 planner를 어떻게 연결하느냐* 에 따라 두 갈래다. 둘 다 'fast/slow 분리'라는 큰 그림은 같지만, interface 설계가 완전히 다른 시스템을 낳는다. Explicit Action Guidance. VLM이 *명시적·읽을 수 있는* 가이드를 내고, 빠른 planner가 그것을 받아 trajectory를 만든다. - Meta-action 가이드: FasionAD는 fast data-driven planner + slow VLM(meta-action 발행) + learned switching. Senna는 Senna-VLM(자연어 결정) + Senna-E2E(실행 trajectory). DiffVLA는 VLM이 lateral·longitudinal 결정을 *one-hot prior*로 diffusion planner에 주입해 multi-modal trajectory denoising을 guidance. - Waypoint supervision: DriveVLM은 chain-of-thought로 coarse waypoint를 먼저 만들고 conventional planner가 dense하게 refine. SOLVE는 공유 vision encoder + 'Trajectory CoT' 모듈로 후보를 반복 정제 후 최종 selection. 장점: 인터페이스가 사람이 읽을 수 있어 디버깅·인증이 쉽다. 약점: VLM 출력이 ambiguous하면 downstream에 그대로 전파. Implicit Representations Transfer. VLM이 *행동에 직접 손대지 않고* 학습 시 teacher 역할만. 두 갈래: - Knowledge distillation: VLP가 BEV feature·planning query를 사전학습 언어 임베딩과 contrastive 정렬, 'commonsense scene understanding'을 planner의 잠재공간에 주입. VLM-AD는 free-form 텍스트 justification + 구조화 behavior label을 alignment head·action classification head로 distill. VERDI·ALN-P3은 perception·prediction·planning 전 단계를 CoT explanation과 정렬하는 *full-stack co-distillation*. - Multimodal feature fusion: InsightDrive는 VLM이 만든 scene description으로 critical region을 강조해 BEV feature를 cross-attention 변조. VLM-E2E는 driver attention cue를 BEV에 gating으로 융합. NetRoller·ReCogDrive는 VLM의 잠재 reasoning을 compact feature로 어댑트. ETA[78]는 efficiency 관점에서 흥미롭다 — VLM reasoning을 이전 frame에서 *비동기로* 계산해 현재 feature에 action-mask로 fuse, 추론 시 VLM을 매 frame 호출할 필요 없게 만든다. 핵심 교훈. Explicit은 *interpretability와 인증*에, Implicit은 *inference latency*와 *VLM 의존성 제거*에 강하다. 둘 다 "VLM이 만들어낸 reasoning이 진짜 행동 quality로 변환되었는가"를 정량화하기 어렵다는 공통 함정도 있다.

Trigger Type C
Explicit Action Guidance
Dual-System VLA에서 VLM이 *사람도 읽을 수 있는* 가이드(meta-action 'change lane', 또는 coarse waypoint sequence)를 내고, downstream fast planner가 이를 받아 dense trajectory를 만드는 구조. 대표: DriveVLM(meta-action→planner refine), Senna(자연어 결정→E2E), DiffVLA(one-hot prior로 diffusion guidance), SOLVE(Trajectory CoT iterative refine). **장점**: interface가 텍스트라 디버깅·인증·human-in-the-loop가 자연스러움. **함정**: VLM hallucination이 그대로 행동에 전파, ambiguous instruction이 downstream cascade 만듦 — Day 2의 cascade 문제가 *다른 층위에서* 재현될 수 있다.
Implicit Representations Transfer (Knowledge Distillation Mode)
VLM을 *runtime이 아닌 학습 시*에만 사용해, reasoning ability를 작은 E2E driving net의 잠재공간으로 옮기는 패러다임. VLP는 BEV+planning query를 언어 임베딩과 contrastive 정렬, VLM-AD는 텍스트 justification을 alignment/classification head로 distill, VERDI·ALN-P3는 perception~planning 전 단계를 CoT와 co-distillation. **장점**: 추론 시 큰 VLM이 필요 없어 latency·메모리 절약, 해석성 일부는 학습된 head로 유지. **함정**: 'oversimplified distillation'으로 reasoning signal이 평탄화되어, VLM이 갖고 있던 long-tail prior가 student로 충분히 전달되지 않을 수 있다 — distill 신호 설계가 성능을 좌우.

객관식 Q1
Dual-System VLA의 *Implicit Transfer* 계열이 가지는 가장 명확한 system-level 이점은?
VLM이 학습 시 teacher로만 쓰이고 추론 시 작은 student만 돌리므로 runtime cost가 결정적으로 줄어듦. 다른 항목은 보장되지 않는다.
객관식 Q2
DiffVLA가 'Explicit Action Guidance'를 구현하는 방식은?
DiffVLA는 VLM의 결정을 명시적 prior로 diffusion planner의 denoising을 가이드 — explicit guidance의 전형.
주관식
내 시스템이 ETA처럼 'VLM을 비동기로 호출'한다면, 가장 큰 실패 모드는 무엇일까? 그것을 마음 속으로 시뮬레이션해 보자.
힌트
VLM의 reasoning이 100ms 전 frame 기준이라면, 그 사이에 갑자기 *cut-in*이 발생했을 때 가이드가 stale해진다. 이 stale guidance가 현재 행동에 어떻게 잘못 영향을 주는지 머릿속에서 한 프레임씩 돌려 보라.
꼭 짚고 가야 할 것
비동기 VLM fusion은 *temporal staleness*가 핵심 리스크다. action-mask·confidence gating으로 stale guidance를 적절히 약화시키는 메커니즘이 안전성의 분기점이 된다.
