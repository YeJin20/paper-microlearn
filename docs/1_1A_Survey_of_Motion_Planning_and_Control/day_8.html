방법론

Control: Pure Pursuit부터 MPC까지

Path stabilization·trajectory tracking·predictive control의 스펙트럼
Motion planner가 reference를 만들면 controller가 그걸 실제로 따라가게 한다. paper는 단순한 것에서 복잡한 것 순으로 정리한다. 1) Pure Pursuit. 차량의 rear axle에서 reference path까지 lookahead distance $L$만큼 떨어진 점을 잡고, 그 점과 차량을 잇는 원호를 따라간다. 곡률 $\kappa = 2\sin\alpha / L$, heading rate $\omega = 2v_r \sin\alpha / L$. DARPA Grand Challenge·Urban Challenge 다수 팀이 채택한 검증된 기법. 약점: 곡률 변화에 정상상태 오차, $L$이 속도 민감 → 보통 $L \propto v_r$로 스케줄. 2) Rear/Front Wheel Position Feedback. Path에서 가장 가까운 점을 찾아 transverse error $e$와 heading error $\theta_e$를 정의. Rear-wheel 기반은 후진에서도 안정($v_r \neq 0$ 어떤 부호든), front-wheel(Stanford Stanley)은 $\delta = \arctan(-ke/v_f) - \theta_e$로 직관적 — 앞바퀴를 path 쪽으로 돌리는 양이 path까지 거리에 비례. 단점: 후진 불가. 3) Control Lyapunov / Output Feedback Linearization. Lyapunov 함수 $V = \frac{1}{2}(x_e^2 + y_e^2) + (1-\cos\theta_e)/k_2$로 안정성 증명. Output linearization은 "앞쪽으로 $d$ 떨어진 점"의 출력을 잡으면 $(\dot{x}_p, \dot{y}_p) = (u_x, u_y)$로 단순화([144]) — 비선형 시스템을 선형으로 보이게 만드는 마법. 4) MPC. 짧은 horizon $N$ 동안 NLP를 풀고 첫 step만 적용한 뒤 다시 푼다. Linear LTV-MPC는 reference 주변 1차 expansion으로 QP화 — interior-point로 다항 시간. paper는 빙판에서 21 m/s, 20 Hz 제어 주파수로 실증된 결과를 인용([148]–[150]). Nonlinear MPC는 정확하지만 실시간 보장 어려움. 5) LPV / Gain Scheduling. 횡 동역학 linearization은 속도 $v_r$이 system matrix에 그대로 들어간다(Eq. V.30). 단일 PD gain은 한 속도에서만 좋다 — closed-loop pole이 $(-k_d \pm \sqrt{k_d^2 - 4k_p v_r})/2$로 속도에 따라 출렁. 해법: $k_p$를 $v_r$의 함수로 스케줄 → LPV controller. 학계는 robust control + convex optimization으로 일반화. 핵심 통찰: 모델이 복잡할수록 stability 증명이 어렵다. Nonlinear MPC는 실무에서 잘 되지만 stability not guaranteed라고 paper가 명시. 이 "실용 vs 보장" 긴장이 학계의 영원한 화두다.

Trigger Type B
Linear Time-Varying MPC with Reference Trajectory Linearization
Nonlinear vehicle dynamics $x_{k+1} = F_k(x_k, u_k)$를 reference trajectory $x_\text{ref}$ 주변에서 1차 Taylor expansion: $\xi_{k+1} = A_k \xi_k + B_k \eta_k$, 여기서 $A_k = \nabla_x F_k(x_\text{ref,k}, u_\text{ref,k})$, $B_k = \nabla_u F_k(\cdot)$, $\xi = x - x_\text{ref}$, $\eta = u - u_\text{ref}$. 결과는 LTV 시스템이라 quadratic objective + polyhedral constraint하에 **QP**가 되며 interior-point로 다항 시간에 풀린다. 왜 작동하나: reference 자체가 dynamically feasible이라면 perturbation $\xi$는 작게 유지되고 1차 근사가 valid. 함정: linearization point가 reference인지(이상적) 현재 operating point인지(robust하지만 myopic)에 따라 stability 성질이 갈린다([148] vs [150]).

객관식 Q1
Pure pursuit controller에서 lookahead distance $L$을 차량 속도에 따라 스케일링하는 가장 정확한 이유는?
$\omega = 2v_r \sin\alpha / L$이므로 $v_r$이 커지면 동일한 $\alpha$에서 $\omega$가 폭증, $L$을 함께 키워 sensitivity를 잡는다.
객관식 Q2
LTV-MPC에서 reference trajectory 주변 linearization이 작동하는 이유로 가장 정확한 것은?
feasible reference이면 closed-loop trajectory가 그 주변에 머물고 $\|\xi\|$가 작으니 1차 근사가 유효. reference가 infeasible이면 이 가정 자체가 깨진다.
주관식
여러분의 시스템이 nonlinear MPC를 채택한다면, stability not guaranteed라는 사실을 production safety case에 어떻게 반영할 것인가?
힌트
Safety case는 실증 데이터로 보강 가능하지만, 어떤 영역(operating envelope)에 한정해서 보장하는지 명시해야 한다. 영역 밖이면 fallback controller로 강제 이양.
꼭 짚고 가야 할 것
이론적 stability 보장이 없어도 production은 가능하지만, 그 영역과 fallback 구조를 명시적으로 정의·검증해야 한다. "실증에서 잘 됐다"만으로는 safety case가 되지 않는다.
