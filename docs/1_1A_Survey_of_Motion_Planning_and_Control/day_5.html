4단 계층의 입출력과 시간 스케일을 머릿속에 고정
오늘은 deep dive 전에 "지도"를 머릿속에 새겨두자. Figure II.1을 다시 보면 4개 블록이 위에서 아래로 흐른다. 1) Route Planning (입력: 도착지 + 도로망, 출력: waypoint sequence). 도로망 그래프는 edge 수가 수백만 단위라 Dijkstra는 비실용 — Contraction Hierarchies 같은 pre-processing 알고리즘이 대륙 규모를 ms 단위로 푼다. 2) Behavioral Decision Making (입력: route + 주변 agent 인지, 출력: "차선 유지/차선 변경/교차로 양보" 같은 motion spec). DARPA 우승팀들은 대부분 FSM(finite state machine)을 썼지만, 보행자 의도 같은 불확실성을 다루기 위해 MDP·POMDP formulation이 학계에선 표준화 중이다. Google은 의도 예측에 학습 기법을 쓴다고 보고됨. 3) Motion Planning (입력: motion spec + 자유 공간 + 자차 상태, 출력: reference path $\sigma: [0,1] \to X$ 또는 trajectory $\pi: [0,T] \to X$). path vs trajectory의 차이는 시간 의존성 — 동적 장애물·동역학을 다루려면 trajectory 필요. PSPACE-hard지만 numerical 방법으로 우회. 4) Feedback Control (입력: reference + 차량 measurement, 출력: steering/throttle/brake). 모델 mismatch와 외란을 흡수하는 마지막 방어선. Pure pursuit ~ LPV-MPC까지 스펙트럼이 넓다. 왜 이 분할이 작동하는가? 각 layer가 다른 시간 스케일에서 도므로 한 layer의 latency가 다른 layer의 정확도를 직접 깎지 않는다. 그리고 layer 간 인터페이스가 단순(waypoint·spec·path·command)해 모듈 교체가 가능하다. 이게 "end-to-end는 통합 학습이 가능하지만 안전 검증이 어렵다"는 현재 학계 논쟁의 출발점이다. 중요한 보너스 정보: paper는 perception을 명시적으로 제외한다. 즉 본 paper의 stack은 perception output(map·obstacle·pose)이 정확하다는 가정 위에 선다. Sim2real에서 이 가정이 깨지는 양상은 perception 논문에서 따로 다뤄야 한다.

Trigger Type B
Path vs Trajectory 구분
Path는 $\sigma(\alpha): [0,1] \to X$로 **순수 기하**만 정의 — 어떻게 그 위를 갈지는 미지정. Trajectory는 $\pi(t): [0,T] \to X$로 **시간 parametrize**되어 속도/가속·동적 장애물·차량 동역학을 모두 표현. 변환은 가능: trajectory planning을 $X^P := X^T \times [0,T]$ 공간의 path planning으로 mapping하고 시간 monotonicity $t(\sigma'(\alpha))>0$을 differential constraint로 추가하면 된다([86]). 함정: trajectory가 더 표현력 있지만, dynamic obstacle이 있으면 path planning(2D)에서 polynomial이던 문제가 trajectory planning(2D+t)에선 NP-hard로 점프한다(Canny & Reif 1987). 즉 시간축을 추가하는 비용이 이론적으로 거대하다.

객관식 Q1
path planning에서 trajectory planning으로 넘어갈 때 발생하는 이론적 비용의 가장 정확한 표현은?
Canny & Reif(1987)가 보였듯 dynamic obstacle 사이 velocity-bounded trajectory 찾기는 NP-hard. 시간축 추가는 표현력과 복잡도를 동시에 끌어올린다.
객관식 Q2
본 paper가 perception을 stack에서 제외한 이유로 가장 적절한 것은?
paper는 명시적으로 "To limit the scope... the broad field of perception is omitted"라며 별도 survey([28]–[31])를 참고하라고 한다.
